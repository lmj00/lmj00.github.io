### 개요  
Cursor AI를 입문으로 생성형 AI를 쓴 지 8개월이 지났다.
그동안 많은 것이 바뀐 것 같다. 나도 Cursor AI를 시작으로 Gemini CLI, Windsurf, Tabnine, Amazon Q, Claude Code, Builder.io를 썼다.

최근에는 Builder.io를 사용해봤는데, 50달러에 500 크레딧이 지급된다.
Figma에서 빌더 플러그인으로 Export 한 코드와 이미지, 요구사항을 주면 피그마 디자인과 거의 비슷하게 주는 것을 보고 정말 놀랐다. 물론 세세한 간격이나 잘못 표현하는 부분도 있지만 프론트 작업을 할 때 사용하면 정말 많은 도움이 될 것 같다.

나는 Claude Code를 주로 쓰고, 보조 느낌으로 Cursor를 사용하는 것 같다.
비용적인 제한이 없으면 Claude, 있으면 Cursor를 사용하는 방향이다. Cursor의 composer 모델보단 아직은 Claude Opus 4.6이 더 낫다고 느낀다. 클로드의 Pro 플랜은 사용량 제한에 너무 빨리 도달하여 Claude Max를 사용하고 있다.

8개월이라는 시간이 짧지도 길지도 않은 시간이지만, 이제 개발자가 직접 코드를 손으로 써가는 시대는 끝났다고 생각한다.
이젠 AI를 관리하는 지휘자가 된 것 같다.
물론 개발은 AI가 해주지만, 예전보다 기초가 더 중요해진 것 같다.
코드 자체는 정말 잘 짜주지만, 과연 이게 맞는지, 개선할 점은 없는지, 잘못된 점은 없는지 확인하고 검증도 할 줄 아는 것도 매우 중요해진 것 같다.

최근엔 클로드에서 super claude, claude skills를 사용하여 업무의 효율을 더 끌어올리고 있다.
[SuperClaude](https://github.com/SuperClaude-Org/SuperClaude_Framework/blob/master/docs/user-guide/commands.md)는 Claude Code에 설치하는 [확장 명령어](https://github.com/SuperClaude-Org/SuperClaude_Framework/blob/master/docs/user-guide/commands.md)모음으로, /sc:명령어 형태로 다양한 기능을 호출할 수 있다. 분석, 리서치, 코드 리뷰, 테스트 등 여러 명령어가 있다.

주로 쓰는 명령어는 /sc:analyze로 코드 품질 분석, /sc:improve 성능 개선, /sc:troubleshoot로 문제 확인, /sc:business-panel로 여러 전문가에게 질문 등이 있는데, 이 중 /sc:business-panel이 가장 인상적이고 재밌었다.

이 명령어는 Clayton Christensen(파괴적 혁신), Michael Porter(경쟁 전략), Peter Drucker(경영 철학) 등의 분야에 대해 페르소나를 주입한 전문가들이 내 질문에 대해 답변을 준다. --mode debate의 옵션을 추가하면 서로 찬반토론을 하여 대립하는데, 이것을 보는 게 상당히 재밌다.
이 명령어를 쓰면 내가 혼자서 질문하는 게 아닌 다른 사람들과 토론하면서 개발하는 것 같고, 나의 관점이 아닌 타인의 관점으로 얘기할 수 있어서 하나의 팀이 된 느낌이었다.

[Claude Skills](https://code.claude.com/docs/ko/skills)로는 SKILL.md 파일을 만들어 Claude의 기능을 확장할 수 있다. 예를 들어 배포, 코드 리뷰, 커밋 등 반복적인 워크플로우를 슬래시 명령어로 만들어두면 직접 사용할 수도 있고, Claude가 상황에 맞게 자동으로 사용한다.

최근 커스텀 명령어를 추가하면서, 문서화를 Claude skills로 하면 좋을 것 같단 생각도 들었다. 문서화를 잘하더라도 관리 포인트가 여러곳으로 나눠지면 유지하기 힘들었기 때문이다.
프로젝트 안에서 문서도 같이 따라가고, 팀원들이 커스텀 명령어를 치면 공통된 답변으로 받으니, 혼란이 없을 것 같았다. 배포방법이나 보일러 플레이트로 해당 프로젝트의 기본적인 코드 생성 방법 같은 것도 넣어두고, 테스트 코드도 기본적인 틀로 생성되니 개발 생산성이 정말 올라간 느낌이 들었다.

이처럼 8개월 전이랑은 비교가 안 될 정도로 AI Agent로 생산성이 눈에 띄게 향상된 것을 느낀다. 지금은 Claude를 사용하고 있지만 나중에는 이를 뛰어넘는 다른 Agent가 나오면 거부감 없이 빠르게 학습하고 사용할 준비를 하는 게 당연해졌다.
앞으로 OpenAI, Google, Anthropic, xAI 등 다양한 회사에서 더 강력한 Agent가 나올 것이다. 어떤 도구가 나오든 빠르게 적응하는 것도 개발자의 역량이라 생각하고, 나도 뒤처지지 않고 열심히 따라가야겠다.
