### 개요

Git 용어집(gitglossary)은 Git에서 사용되는 핵심 개념과 용어들을 체계적으로 정의한 문서로, 저장소 구조, 객체 모델, 브랜치 관리, 병합 방식 등 Git의 기본 메커니즘을 이해하는 데 필수적이다. 개발자가 Git 명령어를 사용할 때 마주치는 다양한 용어들이 실제로 어떤 의미를 가지고 어떻게 동작하는지를 명확히 파악할 수 있다.

### 객체와 저장소 구조

Git의 기본 단위인 **객체(object)**는 내용의 SHA-1 해시값을 기준으로 고유하게 식별되며, 이를 통해 변경 불가능한 저장 방식을 구현한다. 객체는 **커밋 객체**, **트리 객체**, **블로브 객체**, **태그 객체** 네 가지 타입으로 구성되며 각각은 프로젝트의 상태, 디렉터리 구조, 파일 내용, 릴리즈 정보를 담는다.

**객체 데이터베이스(object database)**는 이러한 객체들을 저장하는 공간으로 일반적으로 `$GIT_DIR/objects/` 디렉터리에 위치한다. 또한 **대체 객체 데이터베이스(alternate object database)** 메커니즘을 통해 다른 저장소의 객체를 상속받아 사용할 수 있다.

### 브랜치와 HEAD 개념

**브랜치(branch)**는 프로젝트의 개발 라인을 의미하며, 브랜치의 가장 최근 커밋을 **브랜치 헤드(branch head)**라고 한다. HEAD는 현재 작업 중인 브랜치를 가리키는 참조이지만, 특정 커밋을 직접 체크아웃하면 **분리된 HEAD(detached HEAD)** 상태가 된다.

분리된 HEAD 상태에서는 브랜치 기반의 작업은 불가능하지만, 해당 커밋에서 새로운 커밋을 만들 수 있다. 반면 **깨끗한 작업 트리(clean working tree)**는 현재 HEAD가 가리키는 커밋과 동일한 상태를 의미하며, 수정 사항이 있는 경우 **더티한 작업 트리(dirty working tree)**라고 한다.

### 커밋과 그래프 구조

**커밋(commit)**은 프로젝트 히스토리상의 단일 지점으로, 부모 커밋, 작성자, 날짜, 트리 객체 등의 정보를 포함한다. 커밋들은 서로 부모 관계를 형성하며 **방향성 비순환 그래프(DAG, Directed Acyclic Graph)** 구조를 만든다.

**커밋 그래프 개념**은 이러한 커밋들 간의 관계를 의미하며, **커밋-그래프 파일(commit-graph file)**을 통해 이 구조를 보다 효율적으로 표현할 수 있다. 또한 **커밋-ISH(commit-ish)**는 커밋 객체로 재귀적으로 역참조될 수 있는 객체를 의미한다.

### 인덱스와 체크아웃

**인덱스(index)**는 작업 트리의 스냅샷을 저장하는 공간으로, 파일들의 내용과 메타정보를 포함한다. 인덱스는 병합 시 여러 버전의 파일을 동시에 관리할 수 있다. **체크아웃(checkout)**은 작업 트리를 특정 커밋이나 트리 객체의 상태로 업데이트하는 작업이다.

체크아웃 모드에는 **오버레이(overlay)**와 **논-오버레이(no-overlay)**가 있는데, 전자는 파일을 추가하거나 업데이트만 하고 삭제는 하지 않으며 후자는 대상에 없는 추적 파일도 함께 삭제한다.

### 병합과 머지 전략

**병합(merge)**은 다른 브랜치의 내용을 현재 브랜치에 통합하는 작업으로, 자동으로 변경 사항을 적용하거나 충돌이 발생하면 수동으로 해결해야 한다. **패스트 포워드(fast-forward)**는 병합 대상이 현재 커밋의 후손인 경우 새로운 커밋 없이 브랜치 포인터만 이동하는 특수한 병합 방식이다.

**옥토퍼스(octopus)**는 두 개 이상의 브랜치를 동시에 병합하는 방식이며, **악마의 병합(evil merge)**은 부모 커밋에 존재하지 않는 변경 사항을 도입하는 병합을 의미한다.

### 기타 주요 개념

**후크(hook)**는 Git 명령어 실행 시점에 호출되는 스크립트로, 사전 검증이나 후처리 작업을 자동화할 수 있다. 후크 파일들은 `$GIT_DIR/hooks/` 디렉터리에 위치하며 `.sample` 접미사를 제거하여 활성화한다.

**그라프트(grafts)**는 커밋의 가짜 계보 정보를 기록하여 서로 다른 개발 라인을 연결할 수 있게 하지만, 현재는 **git-replace**가 더 유연하고 견고한 대안으로 권장된다. **패키지(pack)**는 여러 객체를 압축하여 하나의 파일로 저장한 것으로, 저장 공간과 전송 효율성을 높인다.

**패스스펙(pathspec)**은 Git 명령어에서 경로를 제한하기 위한 패턴으로, 와일드카드와 매직 워드를 조합하여 복잡한 경로 매칭 규칙을 정의할 수 있다. 매직 워드에는 `top`, `literal`, `icase`, `glob`, `attr` 등이 있으며 각각 다른 매칭 동작을 수행한다.

### 정리

Git 용어집은 객체, 커밋, 브랜치, 인덱스 등 Git의 핵심 개념들을 체계적으로 정의하여 제공한다. 이를 통해 Git의 불변 객체 모델, DAG 기반 히스토리 구조, 그리고 다양한 병합 전략을 이해할 수 있다. 특히 detached HEAD, fast-forward, pathspec 같은 고급 개념들은 Git 사용 시 자주 마주치는 시나리오에 대한 명확한 이해를 돕는다.
