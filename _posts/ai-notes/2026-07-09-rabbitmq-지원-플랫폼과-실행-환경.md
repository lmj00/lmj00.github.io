> 한 줄 요약: RabbitMQ는 지원되는 Erlang 버전을 제공하는 광범위한 플랫폼에서 실행 가능하며, 주로 Linux 배포판, Windows Server, macOS에서 일반적으로 배포된다.

### 개요
RabbitMQ는 가능한 한 많은 플랫폼에서 실행될 수 있도록 설계되었다. 핵심 요구사항은 **지원되는 Erlang/OTP 버전**이 해당 플랫폼에 존재하는 것이다. 이 원칙 덕분에 멀티코어 서버, 클라우드 배포부터 임베디드 시스템까지 다양한 환경에서 실행될 수 있는 가능성이 열린다. 공식적으로는 Erlang이 지원하는 플랫폼이 RabbitMQ 실행의 기반이 된다.

### 지원 가능한 플랫폼 범위
Erlang 런타임이 지원하는 플랫폼은 RabbitMQ가 이론상 실행될 수 있는 환경을 정의한다. 다음 운영체제들이 이 범주에 포함된다.
* Linux
* Microsoft가 지원하는 Windows 버전 (예: Windows 10)
* Microsoft가 지원하는 Windows Server 버전 (예: Windows Server 2019)
* macOS
* Solaris
* FreeBSD

그러나 **가장 일반적으로 사용되고 배포되는** 오픈소스 RabbitMQ의 환경은 더 구체적이다. 다음 플랫폼들이 실질적인 표준 배포 대상이다.
* Ubuntu 및 Debian 기반 Linux 배포판
* Fedora, RHEL, CentOS 및 RPM 기반 Linux 배포판
* Windows Server
* macOS
* openSUSE Leap

### 상업적 지원 대상 플랫폼
RabbitMQ에 대한 유료 상업적 지원을 구매할 수 있는 플랫폼 목록은 VMware의 공식 페이지에서 별도로 제공된다. 이는 위의 일반적인 지원 범위와는 별개의 정책이다.

### 특정 플랫폼 상세
**Windows**의 경우, 지원되는 Erlang/OTP 릴리스가 실행될 수 있는 모든 데스크톱 및 서버 에디션에서 동작한다. 이는 Windows 10부터 Server 2012, 2022까지를 포함한다.

**다른 UNIX 변종** (FreeBSD, Solaris, NetBSD, OpenBSD 등)의 경우, 공식적으로 지원되지는 않지만 대부분의 POSIX 계층을 가진 시스템에서 Erlang과 RabbitMQ가 실행될 수 있다.

### 가상화 환경과 컨테이너
RabbitMQ는 물리적 하드웨어뿐만 아니라 가상 하드웨어에서도 실행된다. 많은 IaaS(Infrastructure as a Service) 제공자와 컨테이너 환경이 이에 해당한다. 이 원리는 지원되지 않는 플랫폼이 지원되는 플랫폼을 에뮬레이션할 수 있다면 RabbitMQ를 실행할 수 있음을 의미하기도 한다. 또한 여러 회사들이 다양한 클라우드에서 RabbitMQ-as-a-Service를 제공하고 있다.

### 지원되지 않는 플랫폼
일부 플랫폼은 명시적으로 지원되지 않는다.
* z/OS 및 대부분의 메인프레임 시스템
* 메모리가 매우 제한된 시스템 (RAM 100MB 미만)

### 정리
RabbitMQ의 플랫폼 지원 정책은 근본적으로 **Erlang 런타임의 지원 여부**에 달려 있다. 이로 인해 Linux, Windows, macOS, 다양한 BSD 등 광범위한 환경에서 실행 가능성이 보장된다. 실질적인 배포는 주요 Linux 배포판과 Windows Server, macOS에 집중되어 있으며, 가상화 및 컨테이너 환경에서도 잘 동작한다. 다만, 메인프레임이나 극도로 제한된 리소스를 가진 시스템은 공식 지원 대상에서 제외된다.
