> 한 줄 요약: Prometheus는 다차원 데이터 모델과 Pull 방식을 사용하는 자립형 오픈소스 모니터링 및 알림 툴킷으로, 시스템 신뢰성을 중시하지만 100% 정확성이 요구되는 청구 등의 용도에는 적합하지 않다.

### 개요
Prometheus는 SoundCloud에서 시작된 오픈소스 시스템 모니터링 및 알림 툴킷이다. 2012년 시작되어 많은 조직에서 채택되었으며, 2016년 Cloud Native Computing Foundation(CNCF)에 Kubernetes 다음으로 두 번째 호스팅 프로젝트로 합류했다. 이는 프로젝트의 독립적인 거버넌스를 강조한다. Prometheus의 핵심은 **시계열 데이터**를 수집하고 저장하는 것으로, 메트릭 정보는 기록된 타임스탬프와 함께 선택적 키-값 쌍인 **라벨**과 함께 저장된다.

### 핵심 기능과 데이터 모델
Prometheus의 주요 기능은 다음과 같은 특징들로 구성된다.

*   **다차원 데이터 모델**: 시계열 데이터는 **메트릭 이름**과 **키/값 쌍(라벨)** 로 식별된다. 이 다차원성을 활용하기 위해 **PromQL**이라는 유연한 쿼리 언어를 제공한다.
*   **Pull 모델 기반 수집**: 시간대별 수집은 기본적으로 HTTP를 통한 **Pull 모델**로 이루어진다. 단기 작업(short-lived job)의 경우, 중간 **Push Gateway**를 통해 푸시를 지원한다.
*   **자립형 설계**: 분산 저장소에 의존하지 않으며, 단일 서버 노드가 자율적으로 동작한다.
*   **동적 탐색**: 타겟은 서비스 디스커버리 또는 정적 설정을 통해 발견된다.
*   **시각화 지원**: 여러 모드의 그래프 및 대시보딩을 지원한다.

여기서 **메트릭**은 일반적으로 수치적 측정값을 의미한다. 웹 서버의 요청 시간, 데이터베이스의 활성 연결 수 등 애플리케이션마다 측정 대상이 다르다. 이러한 메트릭은 애플리케이션이 특정 방식으로 동작하는 원인을 이해하는 데 중요한 역할을 한다. 예를 들어, 웹 애플리케이션이 느려질 때 요청 수 메트릭을 확인하면 부하 원인을 파악하고 서버 수를 늘리는 결정을 내릴 수 있다.

### 아키텍처와 생태계 구성 요소
Prometheus 생태계는 여러 선택적 구성 요소를 포함한다. 대부분의 컴포넌트는 Go로 작성되어 정적 바이너리로 쉽게 빌드 및 배포된다.

![diagram](/assets/diagrams/2026-07-16-prometheus-시계열-기반의-오픈소스-모니터링-시스템-1.svg)

주요 구성 요소는 다음과 같다.

*   **Prometheus 서버**: 메트릭을 스크랩(수집)하고 시계열 데이터를 로컬에 저장하는 핵심 구성 요소이다. 저장된 데이터에 규칙을 실행해 새로운 시계열을 집계하거나 알람을 생성한다.
*   **클라이언트 라이브러리**: 애플리케이션 코드를 계측(instrumentation)하기 위한 라이브러리.
*   **Push Gateway**: 단기 작업(예: 배치 잡)이 자신의 메트릭을 푸시할 수 있도록 하는 중간 게이트웨이다. Prometheus 서버는 이 게이트웨이로부터 데이터를 풀링한다.
*   **Exporter**: HAProxy, StatsD, Graphite 등과 같은 서비스로부터 메트릭을 추출(Pull)할 수 있도록 하는 특수 목적의 에이전트.
*   **Alertmanager**: Prometheus 서버에서 생성된 알람을 처리(그룹화, 중복 제거, 라우팅)하여 이메일, 슬랙 등으로 전달한다.
*   **Grafana/기타 API 소비자**: 수집된 데이터를 시각화하는 데 사용된다.

### 적용 범위와 한계
Prometheus는 순수한 숫자형 시계열을 기록하는 데 적합하다. 이는 머신 중심 모니터링과 고도로 동적인 서비스 지향 아키텍처(마이크로서비스) 모니터링 모두에 잘 맞으며, 다차원 데이터 수집 및 쿼리 지원이 강점이다.

**신뢰성**을 위해 설계되어, 장애 발생 시 문제를 신속히 진단할 수 있는 시스템으로 사용된다. 각 Prometheus 서버는 독립적이며 네트워크 저장소나 다른 원격 서비스에 의존하지 않아 인프라의 다른 부분이 고장 나도 의존할 수 있다.

그러나 **100% 정확도**가 요구되는 경우, 예를 들어 요청별 청구와 같은 용도에는 적합하지 않다. 수집된 데이터가 충분히 상세하고 완벽하지 않을 가능성이 있기 때문이다. 이러한 경우에는 청구용 데이터 수집 및 분석에는 다른 시스템을, 나머지 모니터링에는 Prometheus를 사용하는 것이 좋다.

### 정리
Prometheus는 자립형 설계와 HTTP 풀 모델을 기반으로 한 강력한 오픈소스 모니터링 툴킷이다. 메트릭 이름과 라벨로 구성된 다차원 데이터 모델과 PromQL을 통해 유연한 데이터 분석을 제공하며, 특히 마이크로서비스 환경에서 강점을 발휘한다. 생태계 내 다양한 컴포넌트(Exporter, Alertmanager, Push Gateway)를 통해 다양한 모니터링 요구사항을 충족시킨다. 핵심 설계 원칙은 장애 상황에서도 사용 가능한 신뢰성에 있으나, 이로 인해 정확성보다는 가용성을 우선시하므로 정확한 청구와 같은 특정 사용 사례에는 제한이 있다.
