### 개요
RabbitMQ 클러스터나 노드가 관리하는 **정의(Definitions)**는 사용자, 가상 호스트, 큐, 익스체인지, 바인딩, 런타임 파라미터와 같은 스키마, 메타데이터, 토폴로지 정보를 말한다. 이 정의를 JSON 파일로 내보내거나 가져오는 기능은 스키마 백업, 클러스터 간 마이그레이션, 배포 시 노드 사전 구성 등에 활용된다. 정의는 내부 데이터베이스에 저장되며 클러스터의 모든 노드에 복제되어 일관성을 유지한다.

## 정의의 특성과 저장 방식
정의는 RabbitMQ의 메타데이터를 총칭한다. 이 정보는 내부 데이터베이스에 저장되며, 클러스터의 **모든 노드에 복제된다**. 정의의 일부가 변경되면, 그 업데이트는 단일 트랜잭션으로 모든 노드에 수행된다. 이 구조 덕분에, 실질적으로 어떤 클러스터 노드에서 정의를 내보내더라도 동일한 결과를 얻을 수 있다.

VMware Tanzu RabbitMQ는 원격 클러스터로의 **Warm Standby Replication**을 지원하여 재해 복구를 위한 대기 클러스터 운영을 용이하게 한다.

## 정의 내보내기 (Export)
정의는 JSON 파일 형식으로 여러 방법을 통해 내보낼 수 있다. 내보내기는 특정 **가상 호스트(vhost)** 단위로 수행하거나 클러스터 전체(모든 가상 호스트)에 대해 수행할 수 있다. 특정 가상 호스트만 대상으로 할 경우, 다른 가상 호스트의 내용이나 대상 가상 호스트에 대한 권한이 없는 사용자 정보는 내보내기 파일에서 제외된다.

내보낸 사용자 데이터에는 비밀번호 해시와 **비밀번호 해싱 함수** 정보가 포함된다. SHA-256이나 SHA-512 같은 함수로 해싱된 비밀번호를 무차별 대입 공격하는 것은 간단한 작업은 아니지만, 사용자 레코드는 **민감 정보로 간주해야 한다**.

주요 내보내기 방법은 다음과 같다.

| 방법 | 필수 플러그인 | 설명 |
| :--- | :--- | :--- |
| `rabbitmqctl export_definitions` | 관리 플러그인 **불필요** | CLI를 통한 기본 방법 |
| `rabbitmqadmin definitions export` | 관리 플러그인 **필요** | HTTP API를 사용하는 관리 CLI |
| `GET /api/definitions` API 엔드포인트 | 관리 플러그인 **필요** | HTTP API 직접 호출 |
| 관리 UI Overview 페이지의 정의 패널 | 관리 플러그인 **필요** | 웹 인터페이스 |

`rabbitmqctl`을 사용한 기본 예시는 다음과 같다.
```bash
rabbitmqctl export_definitions /경로/정의파일.json
```

## 정의 변환 내보내기
`rabbitmqadmin` v2는 정의를 내보내면서 하나 이상의 표준 **변환(transformation)** 함수를 적용할 수 있는 `--transformations` 옵션을 제공한다. 이는 레거시 정책 키 제거, 사용자 정보 난독화, 특정 정의 섹션 제외 등에 유용하다.

사용 가능한 변환 작업은 다음과 같다.

| 변환 이름 | 설명 |
| :--- | :--- |
| `strip_cmq_keys_from_policies` | 모든 내보낸 정책에서 클래식 큐 미러링 관련 키(예: `ha-mode`)를 삭제한다. 이후 `drop_empty_policies` 변환이 따라와야 한다. |
| `drop_empty_policies` | `strip_cmq_keys_from_policies` 후에 사용하여, 키가 모두 제거된 후 정의가 비어 유효하지 않게 된 정책을 제거한다. |
| `obfuscate_usernames` | 사용자명과 비밀번호를 더미 값(`obfuscated-username-1`, `password-1` 등)으로 교체한다. 사용자 및 권한 섹션을 일관되게 업데이트한다. |
| `exclude_users` | 결과에서 모든 사용자를 제거한다. 주로 `exclude_permissions`와 함께 사용된다. |
| `exclude_permissions` | 결과에서 모든 권한을 제거한다. |
| `exclude_runtime_parameters` | 결과에서 모든 런타임 파라미터(연합 업스트림, shovel, Tanzu RabbitMQ의 WSR 및 SDS 설정 포함)를 제거한다. |
| `exclude_policies` | 결과에서 모든 정책을 제거한다. |
| `no_op` | 아무 작업도 하지 않는다. 스크립트 등에서 동적으로 계산된 변환 목록의 기본값으로 사용할 수 있다. |

변환 적용 예시:
```shell
# 클래식 미러링 관련 정책 키를 제거한 후 빈 정책을 삭제하여 표준 출력
rabbitmqadmin definitions export --stdout --transformations strip_cmq_keys_from_policies,drop_empty_policies

# 사용자와 권한을 제거하여 내보내기
rabbitmqadmin definitions export --stdout --transformations exclude_users,exclude_permissions
```

## 정의 가져오기 (Import)
내보내기와 유사하게, `rabbitmqctl import_definitions`, `rabbitmqadmin definitions import`, `POST /api/definitions` API 엔드포인트를 통해 정의 파일을 가져올 수 있다. `rabbitmqctl`을 사용하는 방법은 관리 플러그인이 필요하지 않다는 장점이 있다.

```bash
rabbitmqctl import_definitions /경로/정의파일.json
```

## 노드 부트 시 정의 가져오기
정의 파일을 노드 **시작 시점에 가져오는 것**이 배포 시간에 노드를 사전 구성하는 권장 방법이다. 설정 파일에서 `definitions.import_backend`와 대상 경로를 지정하면 된다. 이 방식은 관리 플러그인을 사전 구성할 필요가 없다.

로컬 파일 시스템에서 가져오는 예시:
```ini
definitions.import_backend = local_filesystem
definitions.local.path = /경로/정의/defs.json
```

HTTPS URL에서 가져오는 예시:
```ini
definitions.import_backend = https
definitions.https.url = https://example.com/definitions.json
```

### 부트 시 가져오기의 세부 사항
정의 가져오기는 **플러그인 활성화 이후**에 발생한다. 따라서 동적 Shovel, 커스텀 타입 익스체인지 등 플러그인과 관련된 정의도 부트 시점에 가져올 수 있다.

정의 파일의 내용은 브로커에 **이미 존재하는 항목을 덮어쓰지 않는다**.
초기화되지 않은 빈 노드가 정의 파일을 가져오면, 기본 가상 호스트(`/`)와 기본 사용자(`guest`)를 생성하지 않는다. **테스트 또는 QA 환경**에서는 정의 파일에 동등한 기본 사용자를 생성하는 JSON 스니펫을 추가할 수 있다. **프로덕션 시스템**에서는 고유한 자격 증명을 가진 새 사용자를 생성하여 사용해야 한다.

### 내용이 변경된 경우에만 가져오기
정의 파일이 자주 변경되지 않는 환경에서는 `definitions.skip_if_unchanged` 설정을 `true`로 하여 **파일 내용이 실제로 변경된 경우에만 가져오기 작업을 수행**하도록 할 수 있다. 이 기능은 로컬 파일, 디렉토리, HTTPS 엔드포인트 가져오기 모두에서 지원된다.

```ini
definitions.skip_if_unchanged = true
definitions.import_backend = local_filesystem
definitions.local.path = /경로/정의/defs.json
```

## 노드 부트 후 정의 가져오기
빌트인 정의 가져오기 기능을 제공하지 않는 이전 버전을 사용하는 경우, 노드 부트 후에 CLI 명령어 조합으로 가져올 수 있다. 먼저 `rabbitmqctl await_startup`으로 노드 시작을 대기한 후, `import_definitions` 명령을 실행한다.

```bash
rabbitmqctl await_startup --timeout 300
rabbitmqctl import_definitions /경로/정의파일.json
```

### 주의사항 및 고려점
* 다중 노드 클러스터에서 부트 시점 가져오기를 사용하면, 모든 노드가 부팅 시 동일한 작업을 **반복적으로 수행**할 수 있다. 정의 파일이 크다면, 클러스터 배포(형성) 후에 부트 후 가져오기 방식을 사용하는 것이 권장된다.
* 내보낸 정의 파일에는 비밀번호 해시가 포함되므로, 파일을 **안전하게 관리**해야 한다.
* 부트 시 가져오기를 사용할 때 정의 파일이 기본 사용자를 생성하지 않음을 인지하고, 환경에 맞는 사용자 생성 전략을 세워야 한다.

### 정리
RabbitMQ의 정의는 시스템 메타데이터의 집합으로, JSON 파일로 손쉽게 내보내고 가져올 수 있다. `rabbitmqctl`(플러그인 불필요)와 `rabbitmqadmin`/HTTP API(플러그인 필요) 두 가지 주요 경로가 있으며, `rabbitmqadmin` v2의 변환 기능을 통해 데이터를 정제할 수 있다. 노드 부트 시 자동 가져오기를 통한 사전 구성이 권장되며, 변경이 적은 환경에서는 `skip_if_unchanged` 설정으로 효율성을 높일 수 있다. 정의 파일은 비밀번호 해시를 포함하므로 민감 정보로 취급하여 관리해야 한다.
